## 진행 기록

- GET /todos 소유자 해석: 2인 구조라 `services/users.resolve_todo_owner`로 대상을 정한다 — 사용자=본인, 관리자=유일한 role='user' 계정. 관리자도 사용자 목록을 조회(SPEC §5)해야 하는데 §3 계약에 user_id 파라미터가 없어 이렇게 해석. e2e로 관리자가 사용자 TODO를 보는 것 확인.
- reorder는 사용자 전용 + 본인의 '오늘' TODO만 + 전부-아니면-거부(부분 적용 방지). 요청 id 중 하나라도 본인/오늘이 아니면 400.
- 토글 포인트 로직은 전부 `services/toggle.py`. A·B 동시 기록은 `_record_both`, 원장 기록/잔액 캐시 갱신은 `services/points.record_point` 한 곳으로 (불변 규칙: 잔액만 갱신 금지). 회수는 `clamp_nonneg=True`로 §4.2 완화 규칙(0까지만 회수, 원장엔 실제 회수액) 반영 → 잔액-원장 항상 일치.
- 완주 중복 지급 방어: 완주 판정 후 daily_bonus_log 행이 이미 있으면 보너스 skip (관리자가 완주 후 TODO를 추가하는 등의 재완주 시 이중지급 차단).
- 라우터에서 잔액 필드를 직접 만지면 post-edit 훅이 불변규칙 위반으로 막는다 → balances를 ToggleResult에 담아 서비스가 반환하도록 수정. 라우터는 얇게 유지.
- Dockerfile이 런타임 의존만 설치해 컨테이너에 pytest/ruff가 없었다(어제 테스트는 호스트에서 돌린 듯). CLAUDE.md의 `docker compose exec api pytest -q`가 바로 되도록 Dockerfile을 `-e ".[dev]"`로 변경 후 재빌드 → 문서화된 명령이 즉시 동작 확인.
